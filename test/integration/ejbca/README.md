# EJBCA integration tests

Integration tests for the CMP client using EJBCA Community Edition.

## Setup and execution

1. Start EJBCA:
   ```bash
   docker compose -f test/integration/ejbca/docker-compose.yml up -d ejbca
   ```

2. Run configuration script:
   ```bash
   ./test/integration/ejbca/setup.sh
   ```

3. Run tests:
   ```bash
   go test -v ./test/integration/ejbca/...
   ```

## Configuration profiles

The tests use custom profiles in `testdata/` imported by `setup.sh`.

### Profile IDs and filenames
EJBCA profile import uses a specific naming convention: `type_NAME-ID.xml`.
The ID suffix is critical because it is used by EJBCA to assign a fixed internal ID during import. This ensures that cross-profile references (e.g., an end entity profile pointing to a certificate profile) remain valid across different EJBCA installations.

### Certificate profile (MYENDUSER)
File: `certprofile_MYENDUSER-2045382623.xml` (ID 2045382623)

Enables Allow Extension Override (set to true). This is required for TestInitializeWithExtensionOverride to allow the client to request custom X.509 extensions.

### End entity profile (MYENDUSER)
File: `entityprofile_MYENDUSER-212939389.xml` (ID 212939389)

This profile explicitly references the certificate profile ID 2045382623 in its internal configuration (fields 29 and 30). It also enables Batch generation (clear text pwd storage) to allow the test harness to set enrollment passwords via the SOAP API.

## Admin interface access

To access the EJBCA Admin Web UI without importing certificates into the browser:

1. Start the nginx proxy:
   ```bash
   docker compose -f test/integration/ejbca/docker-compose.yml --profile nginx up -d nginx
   ```

2. Open http://localhost:8081/ejbca/adminweb/

The proxy uses the `testdata/superadmin-cert.pem` and `testdata/superadmin-key.pem` files for authentication. These are extracted from the container and generated by `setup.sh` during the initialization phase.
